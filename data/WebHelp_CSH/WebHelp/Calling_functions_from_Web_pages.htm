Calling functions from Web pages

In the following examples, note that the API shares the same parameters, regardless of the output type. The only change you need to make is the location of the Help file. (Web pages only support WebHelp, WebHelp Pro, and FlashHelp.)

Syntax:

function RH_ShowHelp(hParent, a_pszHelpFile, uCommand, dwData)

The RH_ShowHelp function syntax has these components:

Parameter

Data Type

Description

hParent

variable

Reserved for future versions of API. Use "0."

a_pszHelpFile

variable

Help source

For WebHelp/FlashHelp: "<Path to project start page>" (Can be local or on a server)

For WebHelp Pro: "http://<ServerName>/roboapi.asp"

*Optional: To specify a window, use ">WindowName" at the end of this parameter

UCommand

variable

Command

You can use the following constants, which are defined in RoboHelp_CSH.js:

HH_DISPLAY_INDEX Displays Index pane and default topic.

HH_DISPLAY_SEARCH Displays Search pane and default topic.

HH_DISPLAY_TOC Displays Contents pane and default topic.

HH_HELP_CONTEXT Opens topic associated with map ID sent in dwData parameter.

*Note: WebHelp Pro/FlashHelp will always open to the default pane, no matter what is specified in the UCommand variable

dwData

variable

Map ID of the topic displayed (defined in the Edit Map IDs dialog in RoboHelp). To obtain the map ID, the author can export the map file for your programming language (you must use HH_HELP_CONTEXT in the uCommand parameter).

 

Tip: To launch the entire Help system (open to the default topic), make a Help call using a uCommand of HH_DISPLAY_INDEX, HH_DISPLAY_SEARCH, or HH_DISPLAY_TOC, and a dwData of "0". Note that in WebHelp Pro/FlashHelp systems, any default window settings for default navigation button will override the HH_DISPLAY constant (however, the Help system will still open with the default topic).

To call a context-sensitive function for WebHelp or FlashHelp:

  1. Call the function.

The following example assumes you have stored the full or relative path to your WebHelp/FlashHelp start page and a window name (optional) in a string variable called "StrPathAndWindow." The value of this string may be something like "WebHelp/MyStartPage.htm>MyWindow". Since you don't know where end users will install your application, this string should never be hard coded.

Note: Ask your technical writer which window(s) to call. Also, if the writer creates the map files, ask the writer to export the map files to the appropriate location.

For example:

To launch the topic associated with map number 1, type the following:

<p>Click for <A HREF='javascript:RH_ShowHelp(0, strPathAndWindow, HH_HELP_CONTEXT, 1)'>Help</A> (map number 1)</p>

  1. Run the application and test the Help call.
